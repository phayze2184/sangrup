# Astro Components Guidelines

## Principles
- Use small, reusable components
- Prefer composition over large files
- Keep logic minimal in .astro files

## Structure
- /components/ui → buttons, inputs, cards
- /components/layout → sections, containers
- /components/blocks → page sections

## Rules
- Always use props for customization
- Avoid inline styles
- Use CSS variables for theming

## When to use
- Creating new UI components
- Refactoring large Astro files
